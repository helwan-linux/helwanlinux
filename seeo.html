Helwan Linux Welcome Tool

The Helwan Linux Welcome Tool is a system management application designed to help users perform common administrative and maintenance tasks without needing advanced terminal knowledge.

This tool focuses on simplicity, safety, and accessibility while still providing powerful system features.

What This Tool Is For

  • Keeping the system up to date
  • Managing Linux kernels
  • Cleaning unused system data
  • Managing essential system services
  • Providing system information
  • Sending feedback to developers

Welcome Tab

The Welcome tab is the main control area of the application. Most users will use this tab frequently.

System Updates

  • Update System (Pacman)
    Updates all official system packages using Pacman. This is the recommended update method for all users.
  • Update System (Yay)
    Updates system and AUR packages using Yay. This option appears only if Yay is installed.

Kernel Management

  • Install Linux Kernel
    Installs or reinstalls the default Linux kernel.
  • Install Linux Zen Kernel
    Installs the Linux Zen kernel, optimized for desktop responsiveness and lower latency.

After installing a new kernel, the bootloader is updated automatically. A reboot is required to use the new kernel.

Application Theme

Allows changing the visual theme of the Welcome Tool. This affects only the application interface, not the system theme.

Application Language

Changes the language used inside the Welcome Tool. Some language changes may require restarting the application.

Startup Behavior

The ‚ÄúShow on startup‚Äù option controls whether the Welcome Tool opens automatically when you log in.

System Language

Allows changing the system-wide language using locale settings. A system reboot is required after applying changes.

System Information

Displays basic hardware and system information, including:

  • Available disk space
  • Processor model
  • Total installed memory (RAM)

System Utilities

  • System Information Details
    Opens a terminal-based system information tool.
  • Performance Monitor
    Opens a terminal tool to monitor CPU and memory usage.

System Cleaner Tab

The System Cleaner tab is used to remove unnecessary files and free disk space safely.

Cleaning Options

  • Clean Pacman Cache (Full)
    Removes all downloaded package files. This action cannot be undone.
  • Remove Orphan Packages
    Removes packages that are no longer required by any installed software.
  • Clean Old Packages
    Removes old package versions while keeping recent ones.

All cleaning actions require administrator privileges. The user will be asked for confirmation before execution.

System Services Tab

The System Services tab provides quick access to essential system services and common maintenance actions.

  • Start important system services
  • Stop services safely
  • Restart services when needed
  • Apply quick fixes for known system issues

This tab is designed to reduce the need for manual systemctl commands.

Feedback Tab

The Feedback tab allows users to communicate directly with the Helwan Linux development team.

  • Report bugs or errors
  • Suggest new features
  • Share general feedback about the system

Feedback is stored temporarily during the session and sent when the user confirms submission.

Security and Permissions

Some actions performed by this tool require administrator (root) privileges.

  • The system will always ask for confirmation
  • No action is executed silently
  • No background changes are made without user approval

Important Notes

  • Always read warnings before performing system actions
  • Kernel changes require a system reboot
  • Language changes may require restarting the application or the system

This tool is designed to be safe for regular users while still providing powerful system management features.

Overview

Momo is a Linux-based **diagnostics tool** designed to help users quickly check the health and performance of their system, including RAM, CPU, storage, and network. It provides a **text-based interactive menu (TUI)** for easy navigation and offers persistent configuration for stress test durations.

The tool is designed for both **quick checks** and **deep diagnostics**, including:

  • RAM and CPU stress testing with configurable durations
  • Disk health and performance checks (including NVMe)
  • Network bandwidth and path quality testing
  • Automatic logging of all test outputs

Note: Some tests require root privileges (especially NVMe and disk tests). Running Momo with sudo is recommended for full functionality.

System Requirements

  • Linux operating system
  • Terminal with enough size (recommended ‚â• 80x24)
  • Installed tools (some are optional):
    • stress-ng (RAM/CPU stress tests)
    • memtester (RAM endurance tests)
    • sysbench (RAM speed test)
    • smartctl (SMART disk tests)
    • fio (Disk IOPS test)
    • iperf3 (Network bandwidth)
    • mtr (Network path quality)
    • nvme-cli (NVMe diagnostics)

Launching Momo

Open a terminal and run:

python3 momo.py

Or, for full disk and NVMe functionality:

sudo python3 momo.py

Interactive Menu

Once launched, Momo presents a scrollable menu where you can select tests using the arrow keys and Enter. Additional commands:

  • A - Run all tests automatically
  • Q - Quit Momo
  • During a test, Q/S - Stop the test early
  • Scroll within long test outputs using ‚Üë and ‚Üì keys

Available Tests

  • RAM Usage: Displays current memory usage.
  • RAM Details: Shows detailed memory info from /proc/meminfo.
  • RAM Stress Test (Short): Quick RAM stress using stress-ng.
  • RAM Endurance Test (Long): Extended RAM stress test for endurance.
  • Memtester 512M (x5): Runs memory tests 5 times.
  • Memory Speed: Measures memory speed using sysbench.
  • Swap Usage: Shows active swap partitions.
  • CPU Info: Detailed CPU information.
  • CPU Stress Test: Stress test for CPU using stress-ng.
  • Smart Status: SMART health info for disks (requires smartctl).
  • Disk IOPS (Fio Test): Measures random I/O performance using fio.
  • Disk Usage: Shows filesystem usage (df -h).
  • Sensors: Displays CPU and motherboard temperatures.
  • Network Ping (Basic): Simple ping test to google.com.
  • Network Bandwidth (iperf3): Measures download/upload bandwidth.
  • Network Path Quality (mtr): Measures latency and packet loss along the network path.
  • NVMe Smart Info: SMART info for NVMe drives (requires nvme-cli and sudo).

Stress Test Configuration

You can configure the duration (in seconds) of stress tests through the menu option "Configure Stress Test Durations". Changes are saved automatically in a persistent configuration file (~/.momo/config.json).

Default Durations:

  • RAM Stress Test (Short): 30 seconds
  • RAM Endurance Test (Long): 3600 seconds (1 hour)
  • CPU Stress Test: 20 seconds

Logs

All test outputs are automatically logged in:

~/.momo/logs

Logs include timestamp, test name, and full command output for later review.

Tips for Users

  • Run Momo in a standard terminal with enough size to avoid display issues.
  • Root privileges are recommended for Disk and NVMe tests.
  • Long stress tests may take a significant amount of time depending on duration settings.
  • Network tests require an active internet connection.

Troubleshooting

  • Missing tool errors: Install the required tool using your package manager (e.g., pacman, apt, dnf).
  • Curses display issues: Ensure your terminal is large enough; resizing may adjust scrollable menus automatically.
  • Permission errors: Use sudo for tests that require access to hardware devices.

Summary

Momo is a **user-friendly, terminal-based diagnostic tool** designed to give both quick insights and deep system tests for Helwan Linux (and compatible Linux distributions). It is ideal for power users and technicians who want:

  • Real-time monitoring
  • Stress and endurance testing
  • Automated logging and historical review

With persistent configuration and scrolling UI, Momo ensures that both short and long-running tests are handled efficiently and safely.

Helwan ISO Signer & Verifier

Helwan ISO Signer & Verifier is a simple tool to sign ISO files and verify their signatures. It is designed for users who want a safe and easy way to generate release files and ensure file integrity.

1. Sign ISO

Follow these steps to sign an ISO file:

  1. Select the ISO File: Click Browse ISO and choose your ISO file.
  2. Select Output Directory: Choose where the signed files will be saved. Default is release folder.
  3. Choose Hash Algorithms: Pick one or more hashing algorithms. Recommended: SHA256.
  4. Start Signing: Click the Sign and Generate Release Files button.
  5. View Progress and Logs: The progress bar shows the overall process. Log messages will appear below.
  6. Final Output: After completion, the output path is displayed. Click Open Folder to view the signed files.
Tip: At least one hash algorithm must be selected before signing.

2. Verify ISO Signature

To verify a signed ISO file:

  1. Select ISO File: Click Browse ISO in the Verify tab.
  2. Select Signature File: Choose the corresponding .sig or .asc file.
  3. Start Verification: Click Start Verification.
  4. Check Results: Logs will show if verification passed or failed. A message box confirms the result.
Note: If verification fails, the ISO file may be corrupted or altered.

3. Hash Options Explained

  • SHA256: Recommended for general security.
  • SHA512: Stronger but slower than SHA256.
  • SHA3-512: Modern and secure alternative.
  • BLAKE2b: Fast and secure hash.
  • SHA1 / MD5: Legacy hashes, less secure, only for compatibility.

4. Tips & Best Practices

  • Always save ISO files in a safe location before signing.
  • Use SHA256 or higher for production releases.
  • Check the output log to ensure all files were signed correctly.
  • Open the final output folder to confirm that signature files were created.

5. Troubleshooting

  • If the Sign button does not respond, make sure the ISO file exists and at least one hash algorithm is selected.
  • If verification fails unexpectedly, ensure you are using the correct signature file for the ISO.
  • Use the log output area to check for any errors during signing or verification.

6. Keyboard Shortcuts

  • Browse ISO: Click the button in each tab.
  • Start Signing: Click Sign and Generate Release Files.
  • Start Verification: Click Start Verification.

7. Support

For issues or feedback, contact Helwan Linux community or visit the official documentation website.

Helwan USB Writer

Helwan USB Writer allows you to safely write ISO files to USB drives and verify their integrity using SHA256 checksums. This guide explains how to use it easily.

1. Select ISO File

  1. Click Select ISO and browse to your ISO file.
  2. The file name will appear under ISO File.
  3. Click Calculate SHA256 to compute the ISO checksum.
  4. The SHA256 value will appear below. This is used for verification later.
Tip: You must calculate the checksum before writing the ISO to USB.

2. Select USB Device

  1. Click Refresh Devices to detect connected USB drives.
  2. Choose your target USB device from the dropdown list.
  3. Warning: All data on the selected USB will be erased!
Make sure you select the correct USB device to avoid losing important data.

3. Write ISO to USB

  1. Once ISO is selected and checksum is calculated, and USB is chosen, click Write ISO to USB.
  2. You may be prompted to enter your password (required by pkexec for safe writing).
  3. Progress and logs will appear in the log section.

4. Verify USB Contents (Optional but Recommended)

  1. After writing completes, you will be asked if you want to verify the USB contents.
  2. If you choose yes, the program will compute the SHA256 checksum of the USB content (up to ISO file size) and compare it to the original ISO.
  3. If the checksums match, the USB is verified and ready.
This step ensures your USB drive is bootable and the ISO was written correctly.

5. Log / Progress

The log area shows:

  • Information messages like scanning devices and starting operations.
  • Progress updates for writing and verification.
  • Errors or warnings if something goes wrong.

6. Safety & Best Practices

  • Always backup important data before writing to a USB drive.
  • Verify the checksum of ISO files before writing.
  • Do not remove the USB during writing or verification.
  • Use the Refresh Devices button if your USB does not appear.

7. Troubleshooting

  • If writing fails, check your password and permissions.
  • If USB verification fails, try reformatting the USB or using another device.
  • Check logs for detailed error messages.

8. Notes

  • pkexec is required on Linux to safely write ISO files to USB.
  • This tool only supports SHA256 verification for ISO and USB.

HelUfw Firewall Manager

HelUfw is an easy-to-use graphical interface for managing your Linux firewall using UFW (Uncomplicated Firewall). You can view status, add/remove rules, reset settings, and control logging.

1. Status Tab

  1. The Status tab shows the current firewall status (Active/Inactive).
  2. Click Refresh Status to update the status and rules.
  3. Use Enable UFW to activate the firewall or Disable UFW to deactivate it.
  4. The output area displays detailed UFW information and any command results.
Tip: After enabling or disabling UFW, always refresh the status to see the updated information.

2. Rules Tab

  1. Add new rules using the "Add New Rule" section:
    • Select action: Allow or Deny.
    • Specify port or port range (e.g., 22, 80, 1024:65535).
    • Choose protocol: TCP, UDP, or leave empty for any.
    • Select direction: in or out, or leave empty for any.
    • Add an optional comment for reference.
    • Click Add Rule to apply it.
  2. The "Existing Rules" section lists all current UFW rules.
  3. Select a rule and click Delete Selected Rule to remove it.
  4. Click Refresh Rules to update the list.
Warning: Rules require administrative privileges. You may be prompted for your password.

3. Settings Tab

  1. Reset UFW: Resets all firewall rules to default. Use with caution, as this cannot be undone.
  2. UFW Logging: Control the firewall logging level:
    • Enable Logging
    • Disable Logging
    • Low Logging Level
Tip: Logging helps you monitor blocked or allowed connections for security auditing.

4. General Notes

  • All actions that modify firewall rules require administrative privileges (you will be prompted for your password).
  • Always refresh the status or rules list after making changes to see accurate information.
  • Be careful when enabling, disabling, or resetting the firewall, as it can affect your system's network connections.

5. Troubleshooting

  • If commands fail, ensure you enter the correct password and that pkexec is installed and configured.
  • If rules are not applied correctly, refresh the rules list to check for updates.
  • For unexpected errors, read the output area for detailed information.

Helwan App Launcher

Purpose

This Python script is the main entry point for the Helwan application. Its job is to:

  • Start the PyQt5 application.
  • Load the application's main window.
  • Apply the Helwan custom styling (QSS) for a consistent look.

How It Works

  1. The script imports necessary modules: sys, os, and PyQt5's QApplication.
  2. It imports the main window class from gui.main_window which contains all UI logic.
  3. It creates a QApplication object, which is required for any PyQt5 GUI application.
  4. It checks if the custom QSS style file helwan_style.qss exists, and applies it to the application if available.
  5. It creates an instance of MainWindow and shows it on the screen.
  6. Finally, it starts the application's event loop using app.exec_().

Files Involved

  • helwan_style.qss: Custom stylesheet for Helwan UI.
  • gui/main_window.py: Contains the main window, menu bar, toolbar, and all UI components.
  • main.py (this script): Launches the application.

Running the Application

To run the Helwan App:

  1. Open a terminal.
  2. Navigate to the project folder.
  3. Run the command:
    4. python3 main.py
  4. The main window should appear with all menus, toolbar, and functionality ready to use.

Notes for Users

- The QSS style file helwan_style.qss is optional. If missing, the app will still run with default PyQt5 styles.
- Closing the window will exit the application.
- This script should not be modified unless you want to change startup behavior or load a different style file.

Summary

This launcher is a simple, safe script that boots your Helwan application and ensures the UI is styled correctly. You don‚Äôt need to interact with it directly besides running it.

Helwan Linux NetFix GUI - User Guide

Purpose

The Helwan NetFix GUI is a graphical tool for Arch-based systems designed to quickly perform maintenance and fixes. It helps users repair the package database, clear caches, fix corrupted packages, update the system, refresh mirrors, and run a full automatic fix.

Main Features

  • Repair Database: Removes unused packages.
  • Clear Cache: Deletes old cached packages.
  • Fix Corrupted Packages: Resolves overwrite or conflict errors.
  • Update System: Synchronizes packages with the latest versions.
  • Refresh Mirrors: Updates package server mirrors for speed and reliability.
  • Full Auto Fix: Runs all maintenance tasks in order automatically.
  • Live Output: Displays command output in a scrollable text area.
  • Progress Bar: Shows ongoing processes visually.
  • Help & About dialogs for guidance.

How It Works

The application uses PyQt5 for its graphical interface. Commands are executed using pkexec to request administrative privileges. The GUI updates in real-time with command output and progress.

Buttons & Commands

  • Each button corresponds to a specific system maintenance task.
  • Clicking a button runs the associated command.
  • While a command is running, the interface is disabled to prevent interference.
  • The text area displays all outputs, including errors.
  • ‚ÄúFull Auto Fix‚Äù runs all commands sequentially, stopping if an error occurs.

Output & Progress

  • The text area shows live output from the commands.
  • Errors are clearly labeled with ERROR:.
  • The progress bar appears during command execution to indicate activity.

Installation Requirements

  • Python 3.x
  • PyQt5 module (pip install PyQt5)
  • pkexec (for graphical privilege escalation)
  • Optional: Ensure your Polkit policies allow pkexec usage in GUI applications.

Running the Tool

  1. Open a terminal.
  2. Navigate to the folder containing hel_netfix_gui_qt.py.
  3. Run the tool with:
    4. python3 hel_netfix_gui_qt.py
  4. The GUI window will open with all buttons, progress bar, and output area.

Help & About

  • Help: Explains what each button does.
  • About: Shows the version, author, and contact information.

Notes for Users

  • Do not close the terminal while commands are running if you launched from a terminal.
  • Ensure pkexec and other necessary tools are installed and in your PATH.
  • The "Full Auto Fix" may stop if a command fails, showing the error in the output area.
  • The tool is safe for Arch-based systems but always back up important data.

Summary

Helwan NetFix GUI provides a safe, easy, and visual way to maintain and fix your Arch-based system without typing complex terminal commands. All operations are logged live for transparency and troubleshooting.

Helwan Blocks - Ultimate User Guide

Welcome!

Helwan Blocks is a modern block-stacking puzzle game (like Tetris) for Helwan Linux. Your goal is to place falling blocks to complete horizontal lines and score points. Clear more lines at once to earn bigger bonuses!

Tip: Focus on planning ahead and keeping your stack flat for maximum score!

Main Menu

  • START GAME: Begin a new game immediately.
  • INSTRUCTIONS: Learn how to play Helwan Blocks and what each key does.
  • HIGH SCORES: View the top scores you or others have achieved.
  • EXIT: Close the game safely.
  • The Helwan Logo and title are just for visual fun and do not affect gameplay.

How to Play

Use the keyboard to control the falling blocks:

  • Left Arrow: Move the block left
  • Right Arrow: Move the block right
  • Down Arrow: Move the block down faster (soft drop)
  • Up Arrow: Rotate the block
  • Spacebar: Instantly drop the block to the bottom (hard drop)
  • C Key: Hold or swap the current block with a stored block
  • Clear full rows to score points. Clearing multiple rows gives extra points!
  • Press M anytime to return to the main menu.
Tip: Using the hold function strategically can help you set up combos!

Game Screen Overview

  • Score Panel: Shows your current score.
  • Next Block: Displays the upcoming block so you can plan ahead.
  • Held Block: Displays the block you have on hold.
  • Playing Field: The area where blocks fall and are stacked.
  • Game Over Screen: Appears when blocks reach the top. You can restart by pressing R or return to menu with M.

Scoring Tips

  • Clear 1 line: small points
  • Clear 2 lines: more points
  • Clear 3 lines: bigger bonus
  • Clear 4 lines at once (a Tetris!): maximum points
  • Combo bonus: clearing lines consecutively gives extra points.

High Scores

  • Access the high scores from the main menu.
  • The top score is highlighted in green.
  • If no scores exist yet, play and your score will appear here!

Game Tips & Tricks

  • Keep the stack as flat as possible to avoid "game over" quickly.
  • Use the hold feature wisely to save crucial blocks for later.
  • Plan ahead using the next block preview.
  • Hard drops are fast and score extra points.
  • Don't panic! Stay calm and keep clearing lines.
Tip: Focus on combos and multiple line clears to achieve high scores!

Exiting the Game

  • From the main menu, click EXIT or press the window close button.
  • Game automatically saves your high scores locally.
